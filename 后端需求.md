Editor（正式编辑）：核心功能是终审判定、全流程统筹、团队权限分配，拥有最高权限，负责稿件全流程最终决策。
Associate Editor（副编辑）：核心功能是稿件初筛、推荐审稿人、审核评审意见，无最终决策权，仅提供专业领域参考建议。
Editorial Assistant & Advisory Editor（EA&AE，编辑助理/顾问编辑统称）：核心功能兼具两部分，一是流程文书处理、进度跟进、作者基础沟通等辅助操作；二是学术质量把控、争议稿件专业咨询，整体以辅助操作为主，无评审与决策权限，仅提供专业建议，不参与日常流程执行。
所有角色均不可自由注册获得，必须走：申请 / 提名 → 审核 → 任命 → 激活。
编辑委员会（Editorial Board，简称 “编委会”）所有编辑角色、权限、流程均由编委会统一管理。
编委会成员 = Editor（正式编辑）
一个编委账号 = 一个 Editor 角色

# 期刊平台极简版API需求

## 设计原则
- **角色通用**：同一接口，不同角色看到不同数据
- **动作聚合**：一个接口处理一类业务的所有操作
- **资源导向**：严格按RESTful资源设计

## 核心亮点

### 1. 稿件流转单接口驱动
`/manuscripts/{id}/workflow` 一个接口处理投稿全流程10+种操作

### 2. 角色权限控制
同一接口，根据token角色返回不同数据：
- GET /manuscripts
  - 作者 → 我的投稿
  - 审稿人 → 分配给我的
  - 编辑 → 待处理的
  - 管理员 → 全部

### 3. 通用查询参数
```
GET /manuscripts?status=pending&assigned_to_me=true
GET /users?role=reviewer&status=active
GET /public/info?type=editorial-board|announcements|guide
```

### 4. 统一响应
```json
{
  "code": 200,
  "message": "success",
  "data": { },
  "meta": {
    "total": 100,
    "page": 1,
    "size": 10
  }
}
```
